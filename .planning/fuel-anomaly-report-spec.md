## 1. חזון המוצר

דוח חריגים דלק הוא **אחד הדוחות החשובים ביותר במערכת ChemoSystem**.
המטרה: לתת למנהל צי הרכב **תמונת מצב ברורה ומיידית** של כל הפעולות
החריגות שבוצעו בחודש נתון — כדי לחקור, ללמוד, ולמנוע נזק כספי עתידי.

### עקרונות מנחים:
- **תוך 3 שניות** — מנהל הצי מבין את החומרה
- **תוך 10 שניות** — מנהל הצי מזהה את הרכבים הבעייתיים
- **תוך 30 שניות** — מנהל הצי יכול לדרוש הסברים מהנהגים

## 2. החלטת ארכיטקטורה: דף נפרד (לא דיאלוג)

### למה דף ולא דיאלוג?
| קריטריון | דיאלוג | דף נפרד ✅ |
|-----------|--------|-----------|
| שטח תצוגה | מוגבל (max-w-5xl) | **מלא** — כל רוחב המסך |
| ניווט ישיר | לא — חייב לפתוח מדף תדלוקים | **כן** — URL ישיר, bookmark |
| כלים (סינון, קיבוץ, חיפוש) | צפוף | **מרווח ונוח** |
| ייצוא + הדפסה | מוגבל | **טבעי** |
| חשיבות עסקית | "חלון צדדי" | **דוח ראשי** — כמו שמגיע לו |

### נתיבים:
```
/app/fleet/exceptions          ← דף בחירת תחום (דלק, אגרה, ...)
/app/fleet/exceptions/fuel     ← דוח חריגים דלק
/app/fleet/exceptions/tolls    ← (עתידי) כבישי אגרה
```

### ניווט:
1. **סיידבר** — "טבלאות חריגים" (שינוי שם מ"טבלת") → `/app/fleet/exceptions`
   - פריט קיים ב-`FleetSidebar.tsx` שורה 60 — `key: "app_fleet_exceptions"`
2. **דף בחירת תחום** → כרטיסיות לבחירה → לחיצה על "תדלוק רכבים" → `/app/fleet/exceptions/fuel`
3. **כפתור בדף תדלוקים** — הכפתור הכתום הקיים → `router.push('/app/fleet/exceptions/fuel')`
4. **URL ישיר** — `/app/fleet/exceptions/fuel?month=2&year=2026`

## 4. פירוט 6 החריגים

### 4.1 🔴 רכב לא מוכר במערכת
- **זיהוי:** `vehicle_id IS NULL`
- **משמעות:** תדלוק עם לוחית שלא קיימת בטבלת `vehicles` — אולי רכב חדש שלא הוזן, שגיאה בלוחית, או תדלוק חשוד
- **חומרה:** קריטית — מישהו תדלק רכב שלא שלנו
- **עמודות בקבוצה:** מס' רישוי | תאריך | שעה | ספק | כמות | סכום נטו | תחנה
- **הערה:** אין יצרן/דגם, אין נהג — כי הרכב לא מוכר

### 4.2 🔴 רכב לא פעיל בזמן התדלוק
- **זיהוי:** `vehicle_status NOT IN ('active', 'suspended')`
- **משמעות:** הרכב רשום במערכת אבל לא במצב פעיל — למה מתדלקים אותו?
- **חומרה:** קריטית
- **עמודות:** מס' רישוי | יצרן/דגם | סטטוס הרכב | תאריך | כמות | סכום נטו | נהג

### 4.3 🔴 רכב מושבת עם חלופי פעיל
- **זיהוי:** `has_active_replacement = TRUE`
- **משמעות:** הרכב המקורי מושבת ויש לו חלופי — מדוע תודלק המקור? הנהג צריך לתדלק את החלופי בלבד
- **חומרה:** קריטית
- **עמודות:** מס' רישוי (מקורי) | רכב חלופי | תאריך | כמות | סכום נטו | נהג | פרויקט
- **הערה:** `original_plate` מה-RPC מספק את הקישור

### 4.4 🟠 חריגת כמות מול מכסה חודשית
- **זיהוי:** `SUM(quantity_liters) > monthly_fuel_limit_liters` לכל מספר רישוי בחודש
- **משמעות:** הרכב צרך יותר דלק ממה שמותר לו — תדלוק מוגזם או שימוש לא מאושר
- **חומרה:** אזהרה — הנפוץ ביותר, לא בהכרח הונאה
- **עמודות:** מס' רישוי | יצרן/דגם | סה"כ ליטרים בחודש | מכסה חודשית | **חריגה (ליטר)** | **% חריגה** | נהג | פרויקט
- **חישוב:** לוגיקה צד-client — סכום ליטרים per plate, השוואה למכסה
- **תצוגה מיוחדת:** שורה אחת לכל רכב (לא לכל תדלוק) — עם אפשרות לפתוח ולראות את כל התדלוקים

### 4.5 🟠 תדלוק כפול באותו יום
- **זיהוי:** `COUNT(*) >= 2` עבור אותו `license_plate + fueling_date`
- **משמעות:** שני תדלוקים או יותר באותו יום — לגיטימי לפעמים (רכב כבד) אבל שווה בדיקה
- **חומרה:** אזהרה
- **עמודות:** מס' רישוי | יצרן/דגם | תאריך | מספר תדלוקים | סה"כ כמות | סה"כ סכום | נהג
- **תצוגה מיוחדת:** שורה אחת לכל (רכב+תאריך), עם אפשרות פתיחה לראות כל תדלוק בנפרד
- **הערה:** שני התדלוקים מוצגים inline כש-row מורחב (שעה + כמות + תחנה)

### 4.6 🟡 תדלוק באזור אילת
- **זיהוי:** `station_name` מכיל אחת מ: אילת, מצפה רמון, יטבתה, באר אורה, ספיר, פארן, עין יהב, חצבה, ערבה, נאות הכיכר, תמנע
- **משמעות:** תדלוק בדרום הרחוק — לא בהכרח חריגה, אבל מסמן רכב שנמצא רחוק מהבסיס
- **חומרה:** לתשומת לב בלבד
- **עמודות:** מס' רישוי | יצרן/דגם | תאריך | כמות | סכום | **שם תחנה** | נהג | פרויקט
- **סגור בדיפולט** — נפתח רק בלחיצה (פחות דחוף)

## 6. פילטרים

### 6.1 בורר חודש
- **מיקום:** header הדף, ליד הכותרת
- **עיצוב:** חיצי ניווט `◄ פברואר 2026 ►`
- **ברירת מחדל:** חודש קודם (כמו היום — מרץ → ברירת מחדל פברואר)
- **התנהגות:** שינוי חודש → קריאה מחדש ל-server action → טעינת נתונים חדשים

### 6.2 סוג חריגה (Multi-select dropdown)
- **אפשרויות:** 6 סוגי חריגה, כל אחד עם ספירה
- **ברירת מחדל:** הכל מסומן
- **עיצוב:** dropdown עם checkboxes, ספירה ליד כל אפשרות

### 6.3 Toggle חומרה
- **עיצוב:** 3 chips צבעוניים: `🔴 קריטי` `🟠 אזהרה` `🟡 לתשומת לב`
- **התנהגות:** toggle — לחיצה מפעילה/מכבה כל רמה
- **ברירת מחדל:** כולם פעילים
- **סינכרון:** לחיצה על KPI card = toggle של אותה חומרה בלבד

### 6.4 חיפוש
- **שדה חיפוש חופשי** — מסנן לפי: מספר רישוי, שם נהג, שם פרויקט
- **filter-as-you-type** עם debounce 300ms
- **client-side** — הנתונים כבר בזיכרון (מקסימום כמה מאות שורות לחודש)

## 9. ייצוא Excel

**כפתור "📥 ייצוא Excel"** ב-header הדף — מייצא את **מה שמוצג** (אחרי סינון).

- אותו פורמט Excel שכבר בנינו ב-Route Handler (צבעים, comments)
- **שינוי:** ה-Route Handler יישאר כ-backend — אבל הלוגיקה תהפוך ל-Server Action
- Excel export = פונקציונליות משנית, הדוח במערכת הוא ה-primary

### לוגיקת זיהוי חריגות (client-side):
```typescript
type AnomalyType =
  | 'unknown_vehicle'        // 🔴 רכב לא מוכר
  | 'inactive_vehicle'       // 🔴 רכב לא פעיל
  | 'active_replacement'     // 🔴 חלופי פעיל
  | 'quantity_exceeded'      // 🟠 חריגת כמות
  | 'duplicate_fueling'      // 🟠 תדלוק כפול
  | 'eilat_area'             // 🟡 אזור אילת

type SeverityLevel = 'critical' | 'warning' | 'info'

type ProcessedAnomaly = {
  type: AnomalyType
  severity: SeverityLevel
  rows: FuelAnomalyRawRow[]        // תדלוקים בקבוצה
  vehicleCount: number              // רכבים ייחודיים
  // -- נתונים ספציפיים לסוג --
  quantityExceeded?: number         // חריגה בליטרים (לסוג 4)
  percentExceeded?: number          // % חריגה (לסוג 4)
  duplicateCount?: number           // מספר תדלוקים כפולים (לסוג 5)
}
```

### RPC — שינוי נדרש:
ה-RPC הקיים `get_fuel_anomaly_report` **מספיק** — הוא מחזיר את כל הנתונים.
ה-`has_active_replacement` כבר קיים ב-type אבל **צריך לוודא שרץ בDB**.

חריגים 4 (חלופי), 5 (כפול), 6 (אילת) — **לוגיקה בצד client בלבד**.
ה-RPC מחזיר raw data, הזיהוי נעשה ב-JS:
- **חלופי פעיל:** `has_active_replacement === true`
- **כפול:** `GROUP BY license_plate, fueling_date` → `count >= 2`
- **אילת:** `station_name.includes(keyword)` עבור רשימת מילות מפתח
