Create `src/components/app/fleet/shared/VehicleFitnessLight.tsx` — `'use client'` component.

**Export `computeVehicleFitnessStatus` helper function:**
```
function computeVehicleFitnessStatus(
  testExpiryDate: string | null,
  insuranceMinExpiry: string | null,
  documentMinExpiry: string | null,
  yellowDays: number
): 'red' | 'yellow' | 'green'
```

Logic (different from driver FitnessLight):
- **Red:** test expired OR any insurance expired (both are critical for road legality)
- **Yellow:** any of the 3 (test, insurance, document) within `yellowDays` of expiry
- **Green:** all clear (no expirations within threshold)
- Null dates = no data = treat as green (not red — vehicle may just not have that record yet)

Date comparison: `new Date(dateStr)` set to midnight, compare against today midnight. Days remaining = `Math.ceil((d - today) / 86_400_000)`. Expired = d <= today.

**Export `VehicleFitnessLight` component:**
Props: `{ testExpiryDate, insuranceMinExpiry, documentMinExpiry, yellowDays, className? }`
Renders a colored dot — same visual pattern as driver `FitnessLight`:
- Red dot: `bg-red-500`
- Yellow dot: `bg-yellow-400`
- Green dot: `bg-green-500`
- Sizing: `h-2.5 w-2.5 rounded-full` (same as driver FitnessLight)
- Add pulse animation on red: `animate-pulse`
- Tooltip text (title attribute): "רכב תקין" / "תוקף מתקרב" / "תוקף פג"

Import `cn` from `@/lib/utils` for className merging.

**Do NOT modify or move the existing `FitnessLight.tsx` in drivers/** — it stays as-is. Two separate components is the correct pattern (per research: "Avoid creating a single mega-generic FitnessLight").
