<task type="auto">
  <name>Task 2: Create vehicles.ts complete CRUD server actions</name>
  <files>src/actions/fleet/vehicles.ts</files>
  <action>
Create `src/actions/fleet/vehicles.ts` with `'use server'` directive. Mirror `drivers.ts` pattern exactly for structure, error handling, and auth guard.

**Auth guard:** `verifyAppUser()` from `@/lib/dal` (NOT verifySession — this is ChemoSys, not admin). Import `createClient` from `@/lib/supabase/server`.

**Import types** from `@/lib/fleet/vehicle-types.ts` (type-only imports for Turbopack).

**Export `ActionResult` type:** `{ success: boolean; error?: string }` (same as drivers.ts).

**Functions to implement (21 total):**

**Vehicle CRUD (6 functions):**
1. `getVehiclesList()` → `Promise<VehicleListItem[]>` — query vehicles with company join, assigned driver name (via drivers→employees join), compute nearest test/insurance/document expiry dates from sub-tables. Filter `deleted_at IS NULL`. Order by `created_at DESC`.
2. `getVehicleById(vehicleId)` → `Promise<VehicleFull | null>` — full vehicle data including all MOT fields + company name join + assigned driver name join + supplier name joins (leasing, insurance, fuel_card, garage from vehicle_suppliers table).
3. `createVehicle(licensePlate, companyId)` → `ActionResult & { vehicleId?: string }` — insert with license_plate + company_id. Check for existing active plate first (`deleted_at IS NULL`). Created_by/updated_by = userId.
4. `updateVehicleDetails(input)` → `ActionResult` — update operational fields only (vehicleType, ownershipType, companyId, isActive, assignedDriverId, notes, leasingCompanyId, insuranceCompanyId, fuelCardSupplierId, garageId). NEVER update MOT fields (those come from mot-sync.ts). `revalidatePath('/app/fleet/vehicle-card/' + vehicleId)`.
5. `softDeleteVehicle(vehicleId)` → `ActionResult` — call `soft_delete_vehicle` RPC with p_id + p_user_id. Revalidate list + card paths.
6. `deleteVehicleWithPassword(vehicleId, password)` → `ActionResult` — check password against `FLEET_ADMIN_PASSWORD` env var (same as drivers.ts pattern). If match, call softDeleteVehicle. Return error if mismatch.

**Vehicle Tests (4 functions):**
7. `getVehicleTests(vehicleId)` → `Promise<VehicleTest[]>` — query vehicle_tests WHERE vehicle_id AND deleted_at IS NULL. Order by test_date DESC.
8. `addVehicleTest(input)` → `ActionResult & { id?: string }` — **INSERT always** (not upsert — test history accumulates, no unique constraint). Fields: vehicle_id, test_date, expiry_date, passed (default true), test_station, cost, notes, file_url, alert_enabled (default TRUE). Revalidate card path.
9. `updateVehicleTest(input)` → `ActionResult` — call `update_vehicle_test` RPC. Revalidate card path.
10. `deleteVehicleTest(testId, vehicleId)` → `ActionResult` — call `soft_delete_vehicle_test` RPC. Revalidate card path.

**Vehicle Insurance (4 functions):**
11. `getVehicleInsurance(vehicleId)` → `Promise<VehicleInsurance[]>` — query vehicle_insurance with **vehicle_suppliers join** for supplier name. WHERE vehicle_id AND deleted_at IS NULL. Order by created_at DESC.
12. `addVehicleInsurance(input)` → `ActionResult & { id?: string }` — insert with insurance_type CHECK constraint (mandatory/comprehensive/third_party), supplier_id FK, policy_number, start_date, expiry_date, cost, file_url, alert_enabled (default TRUE). Revalidate card path.
13. `updateVehicleInsurance(input)` → `ActionResult` — call `update_vehicle_insurance` RPC. Revalidate card path.
14. `deleteVehicleInsurance(insuranceId, vehicleId)` → `ActionResult` — call `soft_delete_vehicle_insurance` RPC. Revalidate card path.

**Vehicle Documents (5 functions):**
15. `getVehicleDocuments(vehicleId)` → `Promise<VehicleDocument[]>` — query vehicle_documents WHERE vehicle_id AND deleted_at IS NULL. Order by created_at DESC.
16. `addVehicleDocument(input)` → `ActionResult & { id?: string }` — insert + call `increment_vehicle_document_name_usage` RPC for autocomplete. alert_enabled default FALSE (mirrors driver docs pattern). Revalidate card path.
17. `updateVehicleDocument(input)` → `ActionResult` — call `update_vehicle_document` RPC. Revalidate card path.
18. `deleteVehicleDocument(docId, vehicleId)` → `ActionResult` — call `soft_delete_vehicle_document` RPC. Revalidate card path. Note: storage file NOT deleted (soft-delete pattern — known deferred cleanup).
19. `getVehicleDocumentNameSuggestions(query)` → `Promise<string[]>` — query vehicle_document_names WHERE name ILIKE '%query%' ORDER BY usage_count DESC LIMIT 10.

**Driver Assignment (2 functions):**
20. `getActiveDriversForAssignment()` → `Promise<DriverOptionForAssignment[]>` — query drivers joined with employees WHERE driver is active AND employee is active. Return id + full name. For Phase 14 dropdown.
21. `assignDriverToVehicle(vehicleId, driverId | null)` → `ActionResult` — update vehicles SET assigned_driver_id. Pass null to unassign. Revalidate both vehicle card and vehicle list paths.

**Revalidation paths:**
- Vehicle card: `/app/fleet/vehicle-card/${vehicleId}`
- Vehicle list: `/app/fleet/vehicle-card`

**Error messages in Hebrew** (same pattern as drivers.ts).

**CRITICAL reminders:**
- vehicle_tests = INSERT always (not upsert) — accumulates history
- vehicle_insurance.supplier_id = FK to vehicle_suppliers — join in getVehicleInsurance
- Soft-delete = RPC only, never direct .update({ deleted_at })
- MOT fields are READ-ONLY in this file — mot-sync.ts handles writing them
- Storage bucket for vehicle files = `fleet-vehicle-documents` (NOT fleet-documents)
