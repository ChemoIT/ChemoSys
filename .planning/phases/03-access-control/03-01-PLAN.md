**2. Create `src/actions/templates.ts`** with three Server Actions following the EXACT pattern from `src/actions/companies.ts`:

- **`createTemplate(prevState, formData)`** — Pattern: verifySession -> extract template fields (name, description) -> `TemplateSchema.safeParse` -> insert into `role_templates` table (set `created_by: session.userId`) -> extract permission matrix from formData (keys like `perm_dashboard`, `perm_companies`, etc. — see step 3 for format) -> parse each as number (0, 1, or 2) -> delete any existing `template_permissions` for this template (defensive) -> insert rows into `template_permissions` for each module where level > 0 (module_key + level) -> writeAuditLog -> revalidatePath('/admin/templates'). Handle error code `23505` on name: return `{ name: ['שם תבנית כבר קיים'] }`.

- **`updateTemplate(id, prevState, formData)`** — Same pattern as `updateCompany` but also: after updating the `role_templates` row, delete ALL existing `template_permissions` for this template_id, then re-insert the new permission set. This is a replace-all pattern (same as employee_role_tags). Audit log both the template update and the permission change.

- **`softDeleteTemplate(id)`** — Identical pattern to `softDeleteCompany`: set `deleted_at`, writeAuditLog, revalidatePath. template_permissions rows are NOT deleted (soft-delete preserves history). The `role_templates` RLS filter on `deleted_at IS NULL` prevents them from showing.

**Permission data extraction from FormData:**
The PermissionMatrixEditor (step 3) writes hidden inputs with names `perm_{module_key}` and values `'0'`, `'1'`, or `'2'`. The module keys are the 9 from the modules seed: `dashboard`, `companies`, `departments`, `role_tags`, `employees`, `users`, `templates`, `projects`, `settings`.

```typescript
const MODULE_KEYS = [
  'dashboard', 'companies', 'departments', 'role_tags',
  'employees', 'users', 'templates', 'projects', 'settings',
] as const

// In createTemplate / updateTemplate:
const permissions = MODULE_KEYS.map(key => ({
  module_key: key,
  level: parseInt(formData.get(`perm_${key}`) as string || '0', 10) as 0 | 1 | 2,
}))

// Delete old permissions then insert new ones
await supabase.from('template_permissions').delete().eq('template_id', templateId)
const permRows = permissions
  .filter(p => p.level > 0) // only store non-zero levels; absence = no access
  .map(p => ({ template_id: templateId, module_key: p.module_key, level: p.level }))
if (permRows.length > 0) {
  await supabase.from('template_permissions').insert(permRows)
}
```

CRITICAL: Use `revalidatePath('/admin/templates')` after each mutation.

export default async function TemplatesPage() {
  await verifySession()
  const supabase = await createClient()

  const { data: templates, error } = await supabase
    .from('role_templates')
    .select('*, template_permissions(module_key, level)')
    .is('deleted_at', null)
