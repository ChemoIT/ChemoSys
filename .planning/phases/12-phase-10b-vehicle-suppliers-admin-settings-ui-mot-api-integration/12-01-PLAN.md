Create `src/actions/fleet/vehicle-suppliers.ts` with these Server Actions:

**1. Type definitions:**
```typescript
export type VehicleSupplier = {
  id: string
  supplier_type: string
  name: string
  contact_name: string | null
  phone: string | null
  email: string | null
  address: string | null
  notes: string | null
  is_active: boolean
  created_at: string
}

export const SUPPLIER_TYPE_LABELS: Record<string, string> = {
  leasing:   'חברת ליסינג',
  insurance: 'חברת ביטוח',
  fuel_card: 'ספק כרטיס דלק',
  garage:    'מוסך',
  other:     'אחר',
}
```

**2. getVehicleSuppliers(type?: string):**
- `verifySession()` guard
- Query `vehicle_suppliers` table, SELECT id, supplier_type, name, contact_name, phone, email, address, notes, is_active, created_at
- If `type` param provided, filter by `.eq('supplier_type', type)`
- Order by supplier_type ASC, name ASC
- Return typed array

**3. createVehicleSupplier(formData: FormData):**
- `verifySession()` guard
- Extract: supplier_type, name, contact_name, phone, email, address, notes
- Validate: name required, supplier_type must be one of SUPPLIER_TYPE_LABELS keys
- Phone handling: attempt `normalizePhone()` from `format.ts`, but if it returns null (landline), store raw value with non-digit chars stripped (suppliers can have landlines — relaxation of mobile-only IRON RULE)
- INSERT via Supabase client (not RPC — RLS allows INSERT for authenticated)
- Set `created_by` and `updated_by` to session user's `authUserId`
- `writeAuditLog()` after success
- `revalidatePath('/admin/vehicle-suppliers')`
- Return `{ success: true, data }` or `{ success: false, error: string }`

**4. updateVehicleSupplier(id: string, formData: FormData):**
- Same pattern as create, but `.update()` + `.eq('id', id)`
- Phone: same normalizePhone logic as create
- Set `updated_by` and `updated_at`
- `writeAuditLog()` + `revalidatePath()`

**5. toggleSupplierActive(id: string, isActive: boolean):**
- `verifySession()` guard
- `.update({ is_active: isActive, updated_by, updated_at })` + `.eq('id', id)`
- `revalidatePath('/admin/vehicle-suppliers')`

**6. deleteVehicleSupplier(id: string):**
- `verifySession()` guard
- **MUST use RPC** `soft_delete_vehicle_supplier` (not direct UPDATE) — project pattern
- `supabase.rpc('soft_delete_vehicle_supplier', { p_id: id, p_user_id: user.authUserId })`
- `writeAuditLog()` + `revalidatePath()`
