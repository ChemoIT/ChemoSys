## Decisions Made

- Rate limiting is in the Server Action file (`auth.ts`), NOT in `proxy.ts` — Edge Runtime resets any module-level Map on every request, making rate limiting ineffective there. Node.js runtime persists the Map across requests within the same instance.
- Rate limit fires BEFORE `signInWithPassword` — this prevents Supabase Auth from being hit at all during a brute-force attack.
- `logout()` excluded from rate limiting — it requires an active session cookie, making brute force impossible.
- Used `crypto.randomBytes(32).toString('base64')` instead of `openssl rand -base64 32` — avoids dependency on openssl availability on Windows.
