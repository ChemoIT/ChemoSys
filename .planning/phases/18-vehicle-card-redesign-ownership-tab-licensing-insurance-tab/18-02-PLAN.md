**Layout and UX:**

The journal has a header section showing the current active rate prominently, followed by the history list, followed by the add button.

**Current active rate display:**
- Find the record where `endDate === null` (active record)
- Display: "עלות נוכחית: ₪{amount.toLocaleString('he-IL')}/חודש" in a highlighted card
- Show "מתאריך: {formatDate(startDate)}" below
- If no active record: show "לא הוגדרה עלות חודשית" in muted text

**History list (costs with endDate !== null):**
- Table or simple list — each row: amount in ₪ | startDate → endDate | edit pencil button
- Use `formatDate()` from `@/lib/format` for all dates (dd/mm/yyyy)
- No delete button — immutable audit trail

**Add / Edit form (inline, toggleable):**
- "הוסף שינוי עלות" button opens the add form
- Edit pencil on a row opens the edit form pre-filled
- Form fields:
  - Amount: `<input type="number" min="0" step="0.01">` — label "סכום חודשי (₪)"
  - Start date: `<FleetDateInput>` — label "תאריך התחלה"
  - End date: `<FleetDateInput>` — label "תאריך סיום (ריק = פעיל)" — optional
- Add form calls `addVehicleMonthlyCost({ vehicleId, startDate, amount })`
- Edit form calls `updateVehicleMonthlyCost({ costId, vehicleId, startDate, endDate, amount })`
- After success: `toast.success(...)` + close form (costs list refreshes via revalidatePath)
- After error: `toast.error(result.error)`

**Loading state:** Use `useTransition` for save operations. Show `<Loader2 className="h-4 w-4 animate-spin">` on save button while pending.

**Important constraints:**
- NO delete button anywhere — cost records are immutable audit trail
- The active record (endDate IS NULL) should be visually distinct (e.g. badge "פעיל", different bg)
- `dir="rtl"` on the root div
- Import FleetDateInput from `@/components/app/fleet/shared/FleetDateInput`
- Import VehicleMonthlyCost type from `@/lib/fleet/vehicle-types`
- Import addVehicleMonthlyCost and updateVehicleMonthlyCost from `@/actions/fleet/vehicle-ownership`
- Import formatDate from `@/lib/format`
- Amount input: format stored as number, display with `toLocaleString('he-IL')`
