**3. Create `supabase/migrations/00004_employee_import_function.sql`:**

A SECURITY DEFINER PostgreSQL function that performs upsert using the partial unique index. This is required because Supabase `.upsert()` cannot use partial indexes (PostgREST limitation — error 42P10).

The function accepts all importable employee fields as parameters and does:
```sql
INSERT INTO employees (...) VALUES (...)
ON CONFLICT (employee_number, company_id) WHERE deleted_at IS NULL
DO UPDATE SET first_name = EXCLUDED.first_name, last_name = EXCLUDED.last_name, ... (all importable fields), updated_by = EXCLUDED.updated_by, updated_at = NOW()
RETURNING id
```

Full function signature from the research file (Pattern 6):
- Parameters: p_employee_number TEXT, p_company_id UUID, p_first_name TEXT, p_last_name TEXT, p_id_number TEXT, p_gender TEXT, p_street TEXT, p_house_number TEXT, p_city TEXT, p_mobile_phone TEXT, p_additional_phone TEXT, p_email TEXT, p_date_of_birth DATE, p_start_date DATE, p_end_date DATE, p_department_id UUID, p_sub_department_id UUID, p_passport_number TEXT, p_citizenship TEXT, p_status TEXT, p_imported_by UUID
- Returns UUID (the employee id)
- LANGUAGE plpgsql, SECURITY DEFINER, SET search_path = public

**4. Add `importEmployeesAction` to `src/actions/employees.ts`:**

This is a two-phase Server Action:
- Phase 1 (preview): When `formData.get('action') === 'preview'` — parse the file, return a summary of what would be imported (new count, update count, error rows) WITHOUT writing to DB.
- Phase 2 (confirm): When `formData.get('action') === 'confirm'` — re-parse the file and execute the upserts via RPC.

The action signature:
```typescript
export async function importEmployeesAction(
  prevState: unknown,
  formData: FormData
): Promise<{
  success: boolean
  phase?: 'preview' | 'complete'
  preview?: { total: number; newCount: number; updateCount: number; errors: string[] }
  result?: { imported: number; updated: number; errors: string[] }
  error?: string
}>
```

Helper functions (from research):
- `cellToDateString(value)`: handles Date objects (`.toISOString().split('T')[0]`), strings, and null
- `cellToString(value)`: trims, returns null for empty
- Gender mapping: `ז` -> `'male'`, `נ` -> `'female'`, else null
- Citizenship: `countryCode` present -> `'foreign'`, null -> `'israeli'`
- Status inference: `end_date` set -> `'inactive'`, null -> `'active'`
- Department lookup: fetch all departments for the company, build `Map<dept_number, uuid>`, lookup per row. `0` means no sub-department.

**Preview phase:**
1. Parse file with ExcelJS (`workbook.xlsx.load(buffer)`, first worksheet)
2. Build parsed rows array
3. Query existing employees by company_id to determine which are new vs update
4. Return counts without writing to DB

**Confirm phase:**
1. Parse file again (file is in FormData on the confirm submission too — OR store preview data in the response and re-submit. Simplest: re-parse from file)
2. For each valid row, call `supabase.rpc('upsert_employee', { ... })`
3. Count successful inserts vs updates
4. Write a single bulk audit log entry: action 'IMPORT', entityType 'employees', newData contains { imported: N, updated: M, company_id }
5. revalidatePath('/admin/employees')
6. Return result summary

**Error handling per row:** If a row fails (RPC error), log the error with the row number, continue processing remaining rows. Return errors array in the result.

CRITICAL: The file is re-parsed from FormData on confirm (the client re-submits the form with the same file + action='confirm'). This avoids storing large parsed data between requests.

export default async function EmployeeImportPage() {
  await verifySession()
  const supabase = await createClient()

  const [companiesRes, departmentsRes] = await Promise.all([
