- **`createEmployee(prevState, formData)`** — Pattern: verifySession -> extract `role_tag_ids` via `formData.getAll('role_tag_ids')` -> delete role_tag_ids from rawData before parsing -> `EmployeeSchema.safeParse` -> insert into `employees` table (empty strings converted to null for optional fields: `department_id || null`, `sub_department_id || null`, `date_of_birth || null`, `start_date || null`, `end_date || null`, `id_number || null`, etc.) -> if success and roleTagIds has items, insert into `employee_role_tags` junction -> writeAuditLog -> revalidatePath('/admin/employees'). - **`updateEmployee(id, prevState, formData)`** — Same pattern as `updateCompany` but also: extract `role_tag_ids` from FormData, update employee record, then delete all existing `employee_role_tags` for this employee and re-insert the new set (replace-all pattern from research). Audit log both the employee update and the role tag change.

- **`softDeleteEmployee(id)`** — Identical pattern to `softDeleteCompany`: set `deleted_at`, writeAuditLog, revalidatePath.

- **`suspendEmployee(id)`** — Set `status: 'suspended'`, `updated_by: session.userId`. Fetch old data for audit. Write audit log with action 'UPDATE'. revalidatePath.

CRITICAL: All empty-string optional fields must be converted to null before DB insert/update (e.g., `input.email || null`). This prevents empty strings from being stored in nullable TEXT columns.

**2. Create `src/app/(admin)/admin/employees/page.tsx`:**

Server component following the CompaniesPage pattern:

```typescript
export default async function EmployeesPage() {
  await verifySession()
  const supabase = await createClient()

  // Parallel fetches for all data needed
  const [employeesRes, companiesRes, departmentsRes, roleTagsRes] = await Promise.all([
    supabase
      .from('employees')
      .select('*, companies(name), departments(name), employee_role_tags(role_tags(name))')
      .is('deleted_at', null)
      .order('created_at', { ascending: false }),
    supabase
      .from('companies')
      .select('*')
      .is('deleted_at', null)
      .order('name'),
    supabase
      .from('departments')
      .select('*')
      .is('deleted_at', null)
      .order('name'),
    supabase
      .from('role_tags')
      .select('*')
      .is('deleted_at', null)
      .order('name'),
  ])

  // Error handling — log but don't crash
  if (employeesRes.error) console.error('[EmployeesPage]', employeesRes.error.message)

  return (
    <div className="space-y-6">
      <div className="flex items-center gap-3">
        <h1 className="text-2xl font-bold text-foreground">ניהול עובדים</h1>
        <Badge variant="secondary">{employeesRes.data?.length ?? 0}</Badge>
      </div>
      <EmployeesTable
        employees={employeesRes.data ?? []}
        companies={companiesRes.data ?? []}
        departments={departmentsRes.data ?? []}
        roleTags={roleTagsRes.data ?? []}
      />
    </div>
  )
}
```

The page fetches employees with joins (company name, department name, role tag names) and all reference data needed for the form dropdowns and filter toolbar.
