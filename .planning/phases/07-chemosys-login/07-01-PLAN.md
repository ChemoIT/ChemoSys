1. **Extract generic `checkRateLimit()` helper.** Replace the existing `checkLoginRateLimit(ip)` function with a generic `checkRateLimit(ip, store)` that takes the rate limit Map as a parameter:
   ```typescript
   function checkRateLimit(ip: string, store: Map<string, RateLimitEntry>): boolean {
     const now = Date.now()
     const entry = store.get(ip)
     if (!entry || now > entry.resetAt) {
       store.set(ip, { count: 1, resetAt: now + RATE_LIMIT_WINDOW_MS })
       return true
     }
     if (entry.count >= RATE_LIMIT_MAX) return false
     entry.count++
     return true
   }
   ```

2. **Update existing `login()` to use `checkRateLimit(ip, loginAttempts)`.** Remove the old `checkLoginRateLimit` function entirely. The `login()` function should call `checkRateLimit(ip, loginAttempts)` where it previously called `checkLoginRateLimit(ip)`.

3. **Add separate `loginAppAttempts` Map** at module level — independent from admin `loginAttempts`. Different attack surfaces must not share rate limit counters:

   ```typescript
   const loginAppAttempts = new Map<string, RateLimitEntry>()
   ```

4. **Add `loginApp()` Server Action** — mirrors `login()` exactly, but redirects to `/app` instead of `/admin/dashboard`. Type: `LoginAppState = { error: string } | null`. Must use the same error messages in Hebrew, the same validation checks, and the same Supabase signInWithPassword call.
   ```typescript
   export async function loginApp(
     _prevState: LoginAppState,
     formData: FormData
   ): Promise<LoginAppState> {
     const headersList = await headers()
     const ip = getClientIp(headersList)
     if (!checkRateLimit(ip, loginAppAttempts)) {
       return { error: 'יותר מדי ניסיונות התחברות. נסה שוב בעוד דקה.' }
     }
     // ... same validation as login() ...
     redirect('/app')
   }
   ```

**Important:** Keep `type LoginState` for admin login and add separate `type LoginAppState` for app login — both are `{ error: string } | null` but keep them as separate types for clarity.
