Create a server component at `src/app/app/page.tsx` (standalone — NOT inside a route group):

1. **Server component** (no "use client" — permissions are read server-side):
   ```typescript
   import { verifyAppUser, getAppNavPermissions } from '@/lib/dal'
   import { redirect } from 'next/navigation'
   import Link from 'next/link'
   import { Truck, HardHat } from 'lucide-react'
   import { cn } from '@/lib/utils'
   ```

2. **Auth guard:** Call `await verifyAppUser()` as the first line — this redirects to `/chemosys` if unauthenticated, blocked, or no `app_*` permissions.

3. **Read permissions:** Call `await getAppNavPermissions()` — returns array of `app_*` module keys. Check for `app_fleet` and `app_equipment` in the array.

4. **Auto-redirect logic** (optimize UX — no unnecessary selection screen):
   - If `hasFleet && !hasEquipment` → `redirect('/app/fleet')`
   - If `hasEquipment && !hasFleet` → `redirect('/app/equipment')`
   - If `!hasFleet && !hasEquipment` → `redirect('/chemosys')` (edge case — shouldn't happen if verifyAppUser passed)
   - If both → show the selection UI

5. **Module selection UI:**
   - Dark background page matching the ChemoSys login feel (`bg-sidebar-bg min-h-screen`)
   - White card centered on screen (`bg-brand-card rounded-xl shadow-lg`)
   - Heading "בחר מודול" centered
   - Two module buttons in a 2-column grid:
     - **צי רכב** — `Truck` icon from lucide-react, links to `/app/fleet`
     - **צמ"ה** — `HardHat` icon from lucide-react, links to `/app/equipment`
   - **Active buttons:** `border-brand-primary text-brand-primary hover:bg-brand-primary/10 cursor-pointer`
   - **Disabled buttons:** `border-border text-muted-foreground opacity-50 cursor-not-allowed pointer-events-none` + "אין גישה" text below the label
   - Use `<Link>` for enabled modules, `<div>` for disabled (do NOT use `<Link>` with `pointer-events-none` — it still allows keyboard navigation)

6. **ModuleButton helper** (inline function, not a separate component):
   ```typescript
   function ModuleButton({ href, label, icon, enabled }: {
     href: string; label: string; icon: React.ReactNode; enabled: boolean
   }) {
     const base = "flex flex-col items-center gap-3 p-6 rounded-xl border-2 transition-colors text-center"
     const activeStyles = "border-brand-primary text-brand-primary hover:bg-brand-primary/10 cursor-pointer"
     const disabledStyles = "border-border text-muted-foreground opacity-50 cursor-not-allowed"

     if (enabled) {
       return (
         <Link href={href} className={cn(base, activeStyles)}>
           {icon}
           <span className="font-semibold text-sm">{label}</span>
         </Link>
       )
     }

     return (
       <div className={cn(base, disabledStyles)} aria-disabled="true">
         {icon}
         <span className="font-semibold text-sm">{label}</span>
         <span className="text-xs">אין גישה</span>
       </div>
     )
   }
   ```

7. **Location rationale:** `src/app/app/page.tsx` is outside any route group. This is intentional for Phase 7 — Phase 8 creates `(app)/layout.tsx` which will wrap this page. Next.js App Router allows adding a route group around an existing page without moving it.

**IMPORTANT:** This is a v2.0 UI page — use the `frontend-design` skill for modern, polished module cards with generous spacing, clear icons, and smooth hover transitions.

**IMPORTANT:** Icon size for module buttons should be large (w-10 h-10 or larger) — these are primary navigation targets for field workers on mobile.

**IMPORTANT:** Both module buttons must ALWAYS be visible (per SUCCESS CRITERIA 2) — disabled ones are grayed, not hidden.
