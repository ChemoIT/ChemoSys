Create `src/components/shared/PageSkeleton.tsx` — a configurable skeleton generator that produces page-level loading skeletons from a simple config object.

**Component signature:**
```tsx
interface PageSkeletonConfig {
  /** Page title placeholder width (px). Default: 96 */
  titleWidth?: number
  /** Stat chips/badges to show below title. Array of widths (px). Empty = no chips. */
  chips?: number[]
  /** Filter bar placeholders. Array of widths (px). Empty = no filter bar. */
  filters?: number[]
  /** Table config. If provided, renders table skeleton. */
  table?: {
    /** Column widths (px) for header + each row */
    columns: number[]
    /** Number of data rows. Default: 10 */
    rows?: number
    /** Show pagination footer. Default: true */
    pagination?: boolean
  }
  /** Card grid config. If provided, renders card grid skeleton (alternative to table). */
  cards?: {
    /** Number of card placeholders */
    count: number
    /** Card height (px). Default: 120 */
    height?: number
    /** Grid columns at each breakpoint */
    cols?: string // Tailwind grid-cols class, default: "grid-cols-1 sm:grid-cols-2 md:grid-cols-3"
  }
  /** Max width class. Default: "max-w-[calc(100%-6cm)]" (matches fleet pages) */
  maxWidth?: string
}

export function PageSkeleton({ config }: { config: PageSkeletonConfig })
```

**Implementation details:**

1. **Always render** the animated shimmer bar at top (copy exact pattern from FuelPageSkeleton):
```tsx
<div className="w-full h-1 bg-muted/40 rounded-full overflow-hidden">
  <div className="h-full w-1/3 bg-sky-500/70 rounded-full animate-[shimmer_1.5s_ease-in-out_infinite]" />
</div>
```

2. **Title section**: Single `<Skeleton>` with configurable width

3. **Chips section** (optional): `flex items-center gap-2 mt-2 flex-wrap` with `rounded-full` Skeleton items

4. **Filters section** (optional): `flex flex-wrap gap-2 items-center` with `rounded-full` Skeleton items at h-9

5. **Table section** (optional): Copy exact table structure from FuelPageSkeleton:
   - Header row with `background: '#F8FAFC'`, `borderColor: '#E8EEF4'`
   - Data rows with `borderBottom: '1px solid #EEF3F9'`
   - Pagination footer with `background: '#FAFCFE'`
   - Columns rendered from `config.table.columns` width array

6. **Cards section** (optional, alternative to table): Grid of card-shaped Skeleton blocks with `rounded-2xl` and `bg-white border`

7. **Always include** the inline `<style>` block for the shimmer keyframe animation

8. **dir="rtl"** on the wrapper div

**Usage example (for documentation in JSDoc):**
```tsx
// List page skeleton (like fuel page)
<PageSkeleton config={{
  titleWidth: 96,
  chips: [112, 96, 112],
  filters: [144, 144, 192, 160, 128, 192],
  table: { columns: [80, 50, 90, 80, 100, 60, 60, 60, 60, 80, 60, 70], rows: 10 }
}} />

// Dashboard skeleton (cards)
<PageSkeleton config={{
  titleWidth: 120,
  cards: { count: 6, height: 140, cols: "grid-cols-1 sm:grid-cols-2 lg:grid-cols-3" }
}} />
```

NOTE: This component is a STARTING POINT. For pages with unique layouts (like vehicle card with tabs), teams will still create custom Skeleton components — but they can use PageSkeleton for the common list/dashboard pattern and save significant time.

**Implementation:**
```tsx
'use client'

import { Loader2 } from 'lucide-react'

export function LoadingIndicator({
  isLoading,
  text = 'מעדכן נתונים...',
  size = 'default'
}: LoadingIndicatorProps) {
  if (!isLoading) return null

  const iconSize = size === 'sm' ? 'h-3 w-3' : 'h-4 w-4'
  const textSize = size === 'sm' ? 'text-xs' : 'text-sm'

  return (
    <div className="flex items-center justify-center gap-2 py-2">
      <Loader2 className={`${iconSize} text-sky-600 animate-spin`} />
      <span className={`${textSize} text-muted-foreground`}>{text}</span>
    </div>
  )
}
```

This is a simple extraction of the inline pattern from FuelRecordsPage.tsx. Mark as `'use client'` since it uses conditional rendering based on a client state prop.
