    1. **RLS Policies** (`supabase/migrations/00002_rls_policies.sql`):

    Enable RLS on ALL tables: companies, departments, role_tags, employees, employee_role_tags, users, user_permissions, role_templates, template_permissions, projects, audit_log, modules.

    Phase 1 approach (per research recommendation): Permissive authenticated-user policies. Business logic enforcement happens in Server Actions, not RLS. This keeps Phase 1 simple.

    For each table with CRUD (companies, departments, role_tags, employees, users, role_templates, projects):
    ```sql
    -- Read: any authenticated user can read non-deleted records
    CREATE POLICY "{table}_select" ON {table}
      FOR SELECT TO authenticated
      USING (deleted_at IS NULL);

    -- Insert: any authenticated user
    CREATE POLICY "{table}_insert" ON {table}
      FOR INSERT TO authenticated
      WITH CHECK (true);

    -- Update: any authenticated user (includes soft delete — the UPDATE that sets deleted_at)
    CREATE POLICY "{table}_update" ON {table}
      FOR UPDATE TO authenticated
      USING (true)
      WITH CHECK (true);
    ```
    NOTE: The UPDATE policy uses `USING (true)` — NOT `USING (deleted_at IS NULL)` — because soft delete is an UPDATE that sets deleted_at. If USING filters out records with deleted_at, the soft-delete UPDATE would silently fail (Pitfall 9 from research).

    For junction tables (employee_role_tags, user_permissions, template_permissions):
    ```sql
    CREATE POLICY "{table}_select" ON {table}
      FOR SELECT TO authenticated USING (true);
    CREATE POLICY "{table}_insert" ON {table}
      FOR INSERT TO authenticated WITH CHECK (true);
    CREATE POLICY "{table}_delete" ON {table}
      FOR DELETE TO authenticated USING (true);
    ```

    For audit_log:
    ```sql
    -- Read only for authenticated users (no write via RLS — writes happen via service role in Server Actions)
    CREATE POLICY "audit_log_select" ON audit_log
      FOR SELECT TO authenticated USING (true);
    -- Insert allowed for authenticated (Server Actions write audit entries)
    CREATE POLICY "audit_log_insert" ON audit_log
      FOR INSERT TO authenticated WITH CHECK (true);
    ```

    For modules:
    ```sql
    CREATE POLICY "modules_select" ON modules
      FOR SELECT TO authenticated USING (true);
    ```
