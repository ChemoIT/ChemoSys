    3. **Auth Actions** (`src/actions/auth.ts`):
    - `'use server'` directive at top
    - `login(prevState: unknown, formData: FormData)`:
      - Extract email and password from formData
      - Create Supabase server client
      - Call `supabase.auth.signInWithPassword({ email, password })`
      - On error: return `{ error: 'מייל או סיסמה שגויים' }`
      - On success: `redirect('/admin/companies')`
    - `logout()`:
      - Create Supabase server client
      - Call `supabase.auth.signOut()`
      - `redirect('/login')`

    4. **DAL** (`src/lib/dal.ts`):
    - Import `cache` from `react`
    - `verifySession = cache(async () => { ... })`:
      - Create Supabase server client
      - Call `supabase.auth.getClaims()` (fast local JWT verification — NOT getUser)
      - If error or no claims: `redirect('/login')`
      - Return `{ userId: claims.sub, email: claims.email }`
    - Export `verifySession`

    5. **Audit Log Utility** (`src/lib/audit.ts`):
    - `writeAuditLog(params: { userId: string; action: 'INSERT' | 'UPDATE' | 'DELETE'; entityType: string; entityId: string; oldData?: Record<string, unknown> | null; newData?: Record<string, unknown> | null })`:
      - Create Supabase server client
      - Insert into `audit_log` table: user_id, action, entity_type, entity_id, old_data (JSONB), new_data (JSONB)
      - Wrap in try/catch — audit log write failure should NOT block the main operation (log warning to console but do not throw)
    - Export `writeAuditLog`

    6. **Utils** (`src/lib/utils.ts`):
    - If not already created by shadcn/ui init, create with the `cn()` utility (shadcn/ui standard):
    ```typescript
    import { type ClassValue, clsx } from "clsx"
    import { twMerge } from "tailwind-merge"
    export function cn(...inputs: ClassValue[]) {
      return twMerge(clsx(inputs))
    }
    ```
    - If shadcn/ui already created this file, leave it as is.
