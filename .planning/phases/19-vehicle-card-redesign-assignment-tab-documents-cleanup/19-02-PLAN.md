    Completely rewrite `VehicleAssignmentSection.tsx`. The old component is a simple driver dropdown — replace it entirely with the new design.

    **Props shape:**
    ```typescript
    type Props = {
      vehicleId: string
      vehicle: VehicleFull
      driverJournal: VehicleDriverJournal[]
      projectJournal: VehicleProjectJournal[]
      onEditingChange?: (isEditing: boolean) => void
    }
    ```

    **Section structure (top to bottom, RTL):**

    **1. Vehicle Category selector** — Radio-style button group (or styled Select):
    - "רכב מחנה" and "רכב צמוד נהג" options
    - Immediate save on selection change (no save button for category itself — just the camp sub-form below)
    - Save by calling updateVehicleDetails({ vehicleCategory: 'camp' | 'assigned' })
    - Show current category as selected on mount (from vehicle.vehicleCategory)

    **2. Camp Responsible sub-form** (shown ONLY when vehicleCategory === 'camp'):
    - Title: "אחראי רכב"
    - Radio-style selector: "אחראי כללי של הפרויקט" | "אחראי רכב אחר"
    - When "אחראי רכב אחר" selected: show two inputs:
      - שם האחראי (text input)
      - טלפון נייד (text input, digits only hint, format hint 05x-xxxxxxx)
    - Save button (styled teal gradient, enabled only when form has unsaved changes)
    - **Dirty tracking:** Compare current form values (campResponsibleType, campResponsibleName, campResponsiblePhone) against original `vehicle` prop values. Call `onEditingChange(true/false)` when dirty state changes.
    - On save: call updateVehicleDetails({ campResponsibleType, campResponsibleName, campResponsiblePhone })
    - On success: toast.success('הפרטים נשמרו') + router.refresh()
    - Phone field: display formatted (using formatPhone from @/lib/format) but save raw value — the server action handles normalizePhone

    **3. Driver Activity Journal** — shown always (camp vehicles may have had past drivers):
    - Title: "יומן נהגים"
    - **Current driver display:** first record in driverJournal where endDate === null. Show driver name in a styled card.
    - **"שינוי נהג" button:** opens an inline form (NOT Dialog — collapse/expand within the tab):
      - Driver select dropdown (fetched via getActiveDriversForAssignment() on mount)
      - Start date (FleetDateInput — import from '@/components/app/fleet/shared/FleetDateInput')
      - "שייך" button → calls assignDriverJournal(vehicleId, driverId, startDate)
      - "סיים שיוך" button (red/outline) → calls endDriverJournal(vehicleId) — shown only if active driver exists
      - Both actions: toast on success/error, router.refresh(), collapse the inline form
    - **History table:** all driverJournal entries (prop is already ordered start_date DESC):
      - Columns: שם נהג | תאריך התחלה | תאריך סיום (or "נוכחי" green badge if endDate === null)
      - Use formatDate() from @/lib/format for date display (dd/mm/yyyy format)
      - If no entries: show "אין שיוכי נהגים" empty state with dashed border

    **4. Project Activity Journal:**
    - Title: "יומן פרויקטים"
    - **Current project display:** first record in projectJournal where endDate === null. Show project name + number in styled card.
    - **"שייך פרויקט" button:** opens inline form:
      - Project select dropdown (fetched via getActiveProjectsForSelect() on mount)
      - Start date (FleetDateInput)
      - "שייך" button → calls assignProjectJournal(vehicleId, projectId, startDate)
      - "סיים שיוך" button (red/outline) → calls endProjectJournal(vehicleId) — shown only if active project exists
      - Both actions: toast on success/error, router.refresh(), collapse the inline form
    - **History table:** all projectJournal entries:
      - Columns: שם פרויקט | מספר פרויקט | תאריך התחלה | תאריך סיום (or "נוכחי" green badge)
      - formatDate() for dates
      - Empty state: "אין שיוכי פרויקטים"

    **Design rules:**
    - dir="rtl" on outermost div
    - All section headers: text-sm font-semibold text-muted-foreground
    - Styled cards: bg-[#F0F5FB] border border-[#C8D5E2] rounded-xl p-4
    - Empty states: dashed border, muted text
    - "נוכחי" badge: `inline-flex items-center px-2 py-0.5 rounded-full text-xs font-semibold` with bg #DCFCE7, color #16A34A, border #BBF7D0
    - Save button: teal gradient when dirty (`background: 'linear-gradient(135deg, #4ECDC4, #3ABFB6)'`), gray when no changes
    - Lucide icons: Shuffle for category, Users for drivers, FolderKanban for projects, Calendar for dates
    - FleetDateInput for all date fields (NOT native date input)
    - formatDate() and formatPhone() for all display formatting

    **Anti-patterns to avoid:**
    - Do NOT use Dialog for inline forms — collapse/expand within the tab
    - Do NOT call assignDriverToVehicle (old function) — use assignDriverJournal instead
    - Do NOT add dirty tracking to journal sections — only the camp sub-form needs onEditingChange
    - Do NOT use native input type="date" — use FleetDateInput

    **Imports needed:**
    - useState, useTransition, useEffect, useCallback from 'react'
    - useRouter from 'next/navigation'
    - toast from 'sonner'
    - Lucide icons as needed (Shuffle, Users, FolderKanban, Calendar, Loader2, ChevronDown, etc.)

    - FleetDateInput from '@/components/app/fleet/shared/FleetDateInput'
    - getActiveDriversForAssignment, updateVehicleDetails, assignDriverJournal, endDriverJournal, assignProjectJournal, endProjectJournal, getActiveProjectsForSelect from '@/actions/fleet/vehicles'
    - formatDate, formatPhone from '@/lib/format'
    - VehicleFull, VehicleDriverJournal, VehicleProjectJournal, DriverOptionForAssignment from '@/lib/fleet/vehicle-types'
