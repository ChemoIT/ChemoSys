Create `src/app/(admin)/api/export/route.ts` -- a GET Route Handler for exporting any admin table to Excel or CSV.

Follow the research pattern exactly (Pattern 5 in 04-RESEARCH.md).

**Implementation:**

```
import { NextRequest } from 'next/server'
import ExcelJS from 'exceljs'
import { verifySession } from '@/lib/dal'
import { createClient } from '@/lib/supabase/server'
```

ALLOWED_TABLES whitelist: `['companies', 'departments', 'employees', 'projects', 'users', 'role_templates']`

HEBREW_TABLE_NAMES map (for worksheet name): `{ companies: 'חברות', departments: 'מחלקות', employees: 'עובדים', projects: 'פרויקטים', users: 'משתמשים', role_templates: 'תבניות הרשאות' }`

**GET handler:**
1. Try verifySession(). If throws, return Response('Unauthorized', { status: 401 })
2. Read `table` and `format` from searchParams. Default format: 'xlsx'
3. Validate table is in ALLOWED_TABLES. If not, return Response('Invalid table', { status: 400 })
4. Create Supabase server client. Select all columns except internal ones:
   - supabase.from(table).select('*').is('deleted_at', null)
5. Create ExcelJS Workbook:
   - workbook.creator = 'ChemoSystem'
   - worksheet = workbook.addWorksheet(HEBREW_TABLE_NAMES[table])
   - worksheet.views = [{ rightToLeft: true }] -- RTL for Hebrew
6. If data has rows:
   - Filter out columns: 'deleted_at', 'created_by', 'updated_by' (internal UUIDs not useful in export)
   - Set worksheet.columns with header=key, width=20
   - addRows(filtered data)
   - Bold the header row: worksheet.getRow(1).font = { bold: true }
7. If format === 'csv':
   - `const csv = await workbook.csv.writeBuffer()`
   - Return Response with Content-Type 'text/csv; charset=utf-8' and Content-Disposition attachment
8. Else (xlsx):
   - `const buffer = await workbook.xlsx.writeBuffer()`
   - Return Response with Content-Type for xlsx and Content-Disposition attachment
   - Use `buffer as Buffer` cast (same established pattern from 02-02)

Filename format: `{table}_export_{YYYY-MM-DD}.xlsx` (or .csv)

**Security:**
- verifySession() is the auth guard -- same as Server Actions
- ALLOWED_TABLES whitelist prevents SQL injection or access to system tables
- No user input goes directly into a query (table name is whitelisted first)
