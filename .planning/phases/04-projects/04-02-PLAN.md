**A) Create `src/app/(admin)/admin/projects/page.tsx`:**

Server Component following the exact pattern from companies/page.tsx:

1. `await verifySession()` as first line
2. Create Supabase client
3. Parallel fetch with `Promise.all`:
   - **Projects query:** `supabase.from('projects').select('*, pm:employees!project_manager_id(id,first_name,last_name,employee_number), sm:employees!site_manager_id(id,first_name,last_name,employee_number), cvc:employees!camp_vehicle_coordinator_id(id,first_name,last_name,employee_number)').is('deleted_at', null).order('created_at', { ascending: false })`
   - **Employees query:** `supabase.from('employees').select('id, first_name, last_name, employee_number, email, id_number, companies(name)').is('deleted_at', null).eq('status', 'active').order('last_name')`
4. Compute `activeCount = projects.filter(p => p.status === 'active').length`
5. Render:
   ```
   <h1>ניהול פרויקטים</h1>
   <Badge variant="secondary">{activeCount} פעילים</Badge>
   <RefreshButton />
   <ProjectsTable projects={projects} employees={employees} />
   ```

Note on Supabase join types: The `.select()` with FK joins returns arrays or objects depending on relationship cardinality. Since each FK is a single reference (not array), Supabase returns `{ id, first_name, last_name, employee_number } | null`. However, TypeScript may infer array type — use the double-cast pattern `(project.pm as unknown as { ... } | null)` if needed (established pattern from Phase 3 user management).
