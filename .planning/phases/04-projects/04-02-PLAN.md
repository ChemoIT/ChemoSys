Create the complete ProjectForm dialog component. This is a large form (~600+ lines) organized in visual sections. Follow the exact patterns from EmployeeForm.tsx.

'use client' directive.

**Props:**
- `open: boolean`
- `onOpenChange: (open: boolean) => void`
- `employees: Employee[]` -- active employees for PM/SM/CVC selectors
- `project?: ProjectRow | null` -- null = create mode, defined = edit mode
- `clocks?: Array<{ clock_id: string }>` -- existing attendance clocks (edit mode)

**State and hooks:**
- useActionState bound to createProject (create mode) or updateProject (edit mode)
- useForm with zodResolver(ProjectSchema) as any (established cast pattern for Zod v4 + RHF)
- useEffect to handle success (close dialog, reset form) and error (set field errors via form.setError)
- useState for attendance clocks array: `[clocks, setClocks]`
- useState for CVC mode: `[cvcIsEmployee, setCvcIsEmployee]` synced with form field
- useState for logo file (File | null) for upload before submit
- Dynamic import of ProjectLocationPicker with ssr: false and loading placeholder (div with animate-pulse, h-[300px])

**Form submission flow (same pattern as EmployeeForm):**
1. Before submitting to Server Action:
   - If logo file exists, upload to Supabase Storage 'client-logos' bucket (browser client, same pattern as employee photo upload), set client_logo_url in formData
   - Serialize attendance_clocks as JSON string into hidden input
   - Boolean fields (pm_notifications, sm_notifications, cvc_is_employee, supervision_notifications, supervision_attach_reports) must be sent as hidden inputs with 'true'/'false' values
2. Submit via formAction from useActionState

**Form sections (rendered as visual groups with Separator or headings):**

- שם פרויקט (name) -- Input, required
- מספר פרויקט (project_number) -- Input, disabled, placeholder "ייווצר אוטומטית" (edit mode shows existing number)

- סטטוס (status) -- Select with 3 options: פעיל / לצפייה בלבד / לא פעיל + hidden input
