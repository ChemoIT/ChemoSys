**1. Create migration `supabase/migrations/00014_projects_rebuild.sql`:**

DROP TABLE IF EXISTS projects CASCADE (safe -- all Phase 4 code was reverted, no production data).

Create SEQUENCE `projects_number_seq START 1`.

CREATE TABLE projects with ALL columns from the research schema:
- Basic: id (UUID PK), name (TEXT NOT NULL), project_number (TEXT NOT NULL DEFAULT ''), open_date (DATE), expense_number (TEXT), description (TEXT), project_type CHECK ('project','staging_area','storage_area'), status CHECK ('active','view_only','inactive') DEFAULT 'active'
- PM: project_manager_id (UUID FK employees), pm_email, pm_phone, pm_notifications (BOOLEAN DEFAULT TRUE)
- SM: site_manager_id (UUID FK employees), sm_email, sm_phone, sm_notifications (BOOLEAN DEFAULT TRUE)
- CVC: camp_vehicle_coordinator_id (UUID FK employees), cvc_is_employee (BOOLEAN DEFAULT TRUE), cvc_phone
- Client: client_name, client_logo_url
- Supervision: supervision_company, supervision_contact, supervision_email, supervision_phone, supervision_notifications (BOOLEAN DEFAULT FALSE), supervision_attach_reports (BOOLEAN DEFAULT FALSE)
- Location: latitude (DECIMAL), longitude (DECIMAL), radius (INTEGER DEFAULT 100)
- Universal: created_at, updated_at, created_by, updated_by, deleted_at

Create UNIQUE INDEX projects_number_active ON projects(project_number) WHERE deleted_at IS NULL AND project_number != ''.

Create updated_at trigger (uses existing set_updated_at function).

Create generate_project_number() trigger function: IF NEW.project_number = '' OR IS NULL THEN set to 'PR26' || LPAD(nextval('projects_number_seq')::TEXT, 6, '0'). Create BEFORE INSERT trigger.

CREATE TABLE attendance_clocks: id (UUID PK), project_id (UUID FK projects ON DELETE CASCADE NOT NULL), clock_id (TEXT NOT NULL), created_at, created_by. UNIQUE INDEX on (project_id, clock_id).

RLS policies for both tables (same pattern as other admin tables -- permissive for authenticated). attendance_clocks also needs DELETE policy (for replace-all pattern).

Create soft_delete_projects RPC function: SECURITY DEFINER, accepts p_ids UUID[], UPDATE projects SET deleted_at=NOW(), updated_at=NOW() WHERE id=ANY(p_ids) AND deleted_at IS NULL. RETURN affected count. Same pattern as soft_delete_employees in 00007.

Storage: Create 'client-logos' bucket policies (INSERT for authenticated, SELECT public). Follow pattern from 00010_storage_policies.sql. Note: bucket creation must be done in Supabase Dashboard -- SQL can only create policies.

**1. Add ProjectSchema to `src/lib/schemas.ts`:**

Follow existing pattern (EmployeeSchema). All fields use `.optional().or(z.literal(''))` for optional strings. Key fields:
- name: z.string().min(1, 'שם פרויקט הוא שדה חובה')
- project_number: z.string().optional().or(z.literal('')) -- auto-generated, not user input
- open_date: z.string().optional().or(z.literal(''))
- expense_number, description: optional strings
- project_type: z.enum(['project','staging_area','storage_area']).optional()
- status: z.enum(['active','view_only','inactive']).default('active')
- PM fields: project_manager_id (uuid optional), pm_email, pm_phone, pm_notifications (z.boolean().default(true))
- SM fields: same pattern as PM
- CVC fields: camp_vehicle_coordinator_id (uuid optional), cvc_is_employee (z.boolean().default(true)), cvc_phone
- Client: client_name, client_logo_url
- Supervision: supervision_company, supervision_contact, supervision_email, supervision_phone, supervision_notifications (z.boolean().default(false)), supervision_attach_reports (z.boolean().default(false))
- Location: latitude (z.coerce.number().optional().nullable()), longitude (same), radius (z.coerce.number().int().min(0).default(100))
- attendance_clocks: z.string().optional().or(z.literal('')) -- JSON string, parsed in Server Action

Add `.superRefine()` for CVC conditional validation: when cvc_is_employee is false AND cvc_phone is provided, validate against Israeli mobile regex `/^0(5[0-9])[0-9]{7}$/`. When cvc_is_employee is false AND cvc_phone is empty, add required error on cvc_phone.

Export `ProjectInput = z.infer<typeof ProjectSchema>`.

**2. Create `src/actions/projects.ts`:**

Follow the exact pattern from `src/actions/employees.ts`:
- 'use server' directive
- Imports: revalidatePath, createClient, verifySession, writeAuditLog, ProjectSchema

**createProject(prevState, formData):**
- verifySession()
- Extract attendance_clocks JSON string from formData, then delete it from rawData
- Parse boolean fields from formData: pm_notifications, sm_notifications, cvc_is_employee, supervision_notifications, supervision_attach_reports (formData sends 'true'/'false' strings or checkbox 'on' -- convert to boolean before Zod parse)
- ProjectSchema.safeParse(processedData) -- return field errors on failure
- Coerce empty strings to null for all optional fields (same pattern as employees)
- Insert into projects with project_number: '' (trigger fills it in), created_by: session.userId
- Use .select().single() to get back the generated project_number and id
- Parse clocksJson: JSON.parse() to get string array. Filter out empties. Insert into attendance_clocks with project_id and created_by
- writeAuditLog with action: 'INSERT', entityType: 'projects'
- revalidatePath('/admin/projects')
- Return { success: true }

**updateProject(prevState, formData):**
- Same validation flow
- Extract project id from formData
- Update the project row (updated_by: session.userId)
- Replace-all clocks: DELETE existing clocks for this project_id, then INSERT new ones (same pattern as employee_role_tags)
- writeAuditLog with action: 'UPDATE' (pass old data)
- revalidatePath('/admin/projects')

**softDeleteProject(id: string):**
- verifySession()
- Call supabase.rpc('soft_delete_projects', { p_ids: [id] })
- writeAuditLog with action: 'DELETE'
- revalidatePath('/admin/projects')
- Return { success: true }

Important: Use `ActionState` type pattern from employees.ts. All error messages in Hebrew.
