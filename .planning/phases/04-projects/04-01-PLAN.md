1. **Sequence** `project_number_seq` — monotonically increasing, START WITH 1, INCREMENT BY 1, CACHE 1. No reset per year.

2. **`generate_project_number()` function** — SECURITY DEFINER, LANGUAGE plpgsql, SET search_path = public. Logic:
   - `nextval('project_number_seq')` for the sequence number
   - `to_char(CURRENT_DATE, 'YY')` for the 2-digit year
   - Returns `'PR' || year_part || LPAD(next_val::TEXT, 6, '0')` — e.g., `PR26000001`

3. **`soft_delete_project(p_id UUID)` function** — SECURITY DEFINER, LANGUAGE plpgsql, SET search_path = public. Logic:
   - `UPDATE projects SET deleted_at = NOW(), updated_at = NOW() WHERE id = p_id AND deleted_at IS NULL`
   - RETURNS VOID

Follow the exact pattern from `00007_soft_delete_rpc.sql`. Include a header comment block explaining purpose and phase.

Note: This migration file is created locally. Sharon must run it in Supabase SQL Editor before testing the project CRUD flow.

Add after the existing TemplateSchema. Fields:

- `display_name`: `z.string().optional().or(z.literal(''))`

- `has_attendance_code`: `z.string().transform(v => v === 'true')` — hidden input pattern for booleans
- `project_type`: `z.enum(['project', 'staging_area', 'storage_area']).optional().or(z.literal(''))`
- `ignore_auto_equipment`: `z.string().transform(v => v === 'true')` — hidden input pattern

- `pm_email`: `z.string().email('מייל לא תקין').optional().or(z.literal(''))`
- `sm_email`: `z.string().email('מייל לא תקין').optional().or(z.literal(''))`

Export `ProjectSchema` and `type ProjectInput = z.infer<typeof ProjectSchema>`.

**B) Create `src/actions/projects.ts`:**

Follow the exact pattern from `src/actions/companies.ts`. Include `'use server'` header, import verifySession, createClient, writeAuditLog, ProjectSchema.

Three Server Actions:

1. **`createProject(prevState, formData)`** — Returns `{ success: boolean; error?: Record<string, string[]> }`.
   - `await verifySession()`
   - Parse with `ProjectSchema.safeParse(Object.fromEntries(formData))`
   - Call `supabase.rpc('generate_project_number')` to get the auto-number
   - Convert empty string FKs to null: `project_manager_id || null`, `site_manager_id || null`, `camp_vehicle_coordinator_id || null`
   - Convert empty string coordinates to null: `latitude === '' ? null : Number(latitude)`, same for longitude
   - Convert empty string project_type to null
   - Insert into `projects` with `project_number: projectNumber, created_by: session.userId`
   - Handle unique constraint error (23505) on project_number (retry or show error)
   - writeAuditLog with entityType 'projects'
   - `revalidatePath('/admin/projects')`

2. **`updateProject(id, prevState, formData)`** — Same pattern as updateCompany.
   - Never pass `project_number` in the update payload — it is immutable after creation
   - Same empty-to-null conversions as createProject
   - writeAuditLog with old and new data

3. **`softDeleteProject(id)`** — Returns `{ success: boolean; error?: string }`.
   - MUST use `supabase.rpc('soft_delete_project', { p_id: id })` — NOT direct UPDATE (mandatory RPC pattern per 00007)
   - writeAuditLog with action 'DELETE'
   - `revalidatePath('/admin/projects')`
