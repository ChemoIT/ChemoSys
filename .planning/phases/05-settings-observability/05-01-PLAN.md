Create the dashboard page following the established Server Component pattern (see companies/page.tsx).

**1. `src/app/(admin)/admin/dashboard/page.tsx` (Server Component)**
- `await verifySession()` as first line
- Create Supabase client
- Run 7 parallel queries via `Promise.all`:
  - 6 count queries using `.select('id', { count: 'exact', head: true }).is('deleted_at', null)` on tables: `employees`, `projects`, `users`, `companies`, `departments`, `role_tags`
  - 1 audit_log query: `.select('id, created_at, action, entity_type, entity_id, user_id').order('created_at', { ascending: false }).limit(20)`
- For user display names in the activity feed: after fetching audit entries, collect distinct `user_id` values, query `public.users` table with `.select('auth_user_id, full_name, email').in('auth_user_id', distinctUserIds)` to get display names. Build a `Map<string, string>` of user_id -> display name. Merge into the activity entries before passing to ActivityFeed. This two-step approach is needed because `audit_log.user_id` references `auth.users(id)` while display names live in `public.users.auth_user_id` (see Pitfall 2 in RESEARCH.md).
- Pass stats object `{ employees, projects, users, companies, departments, roleTags }` to `<StatsCards>`
- Pass activity entries array (with resolved user names) to `<ActivityFeed>`
- Include `<RefreshButton />` in the page header
- Page header: `<h1>` with "דשבורד" text

**2. `src/components/admin/dashboard/StatsCards.tsx` (Client Component)**
- `'use client'`
- Props: `{ stats: { employees: number, projects: number, users: number, companies: number, departments: number, roleTags: number } }`
- Render 6 cards in a responsive grid: `grid-cols-2 md:grid-cols-3 xl:grid-cols-6 gap-4`
- Each card uses `<Card>` from shadcn/ui with:
  - An icon from lucide-react (Users, FolderKanban, UserCog, Building2, Network, Tags)
  - A Hebrew label (עובדים פעילים, פרויקטים פעילים, יוזרים, חברות, מחלקות, תגיות תפקיד)
  - A large number formatted with `toLocaleString('he-IL')`
  - Distinct color per icon (blue, green, purple, orange, teal, pink)
- Use the STAT_ITEMS config array pattern from RESEARCH.md

**3. `src/components/admin/dashboard/ActivityFeed.tsx` (Client Component)**
- `'use client'`
- Props: entries array with shape `{ id, created_at, action, entity_type, entity_id, user_id, userName }[]`
- Render a vertical list of recent activity entries
- Each entry shows: user name (or truncated UUID if unknown), action (INSERT/UPDATE/DELETE with Hebrew label and color badge), entity type (Hebrew mapped if possible), relative time (use `formatDistanceToNow` from date-fns with `{ locale: he, addSuffix: true }`)
- Action labels: INSERT -> "יצירה" (green badge), UPDATE -> "עדכון" (blue badge), DELETE -> "מחיקה" (red badge), LOGIN -> "כניסה" (gray badge), LOGOUT -> "יציאה" (gray badge)
- Entity type display: use a simple mapping for known types (employees -> "עובד", companies -> "חברה", departments -> "מחלקה", projects -> "פרויקט", users -> "משתמש", role_tags -> "תגית תפקיד", role_templates -> "תבנית הרשאות", employee_import -> "ייבוא עובדים") — fallback to raw entity_type string for unknown types
- Wrap in a Card with header "פעילות אחרונה"
- Handle empty state: "אין פעילות מתועדת" message
- Import `he` locale from `date-fns/locale/he`
