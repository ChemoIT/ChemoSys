**2. `src/lib/env-settings.ts` (server-only library)**

Follow RESEARCH.md Pattern 3 exactly:
- `import 'server-only'` as first line
- `import * as fs from 'fs/promises'`
- `import * as path from 'path'`
- `const ENV_PATH = path.join(process.cwd(), '.env.local')` — ALWAYS .env.local, never .env (Pitfall 3)
- `readEnvFile()`: reads .env.local, parses key=value lines, skips comments and blank lines, strips quotes from values. Returns `Record<string, string>`. Returns empty object if file doesn't exist.
- `writeEnvValues(updates: Record<string, string>)`: reads current file, merges updates, writes back all key=value pairs. Quote values that contain spaces. After writing file, mutate `process.env` in-memory for immediate effect: `(process.env as Record<string, string>)[key] = val` for each key in updates (see Pitfall 4 for TypeScript workaround).
- `deleteEnvKeys(keys: string[])`: reads current file, removes specified keys, writes back. Also `delete process.env[key]` for each removed key.

**3. `src/actions/settings.ts` (Server Actions)**

- `'use server'`
- Import `verifySession` from `@/lib/dal`
- Import `readEnvFile`, `writeEnvValues` from `@/lib/env-settings`
- Import `revalidatePath` from `next/cache`

**`getIntegrationSettings()`**: Reads .env.local and returns structured settings object:
```typescript
{
  sms: { enabled, token, fromName },
  whatsapp: { enabled, accessToken, phoneNumberId, businessAccountId },
  ftp: { enabled, protocol, host, port, username, password, remotePath },
  telegram: { enabled, botToken, chatId },
  llm: { enabled, provider, modelName, apiKey, baseUrl },
}
```
Map env keys to each field (see RESEARCH.md Integration Fields Reference for exact key names). Boolean `enabled` fields: `env['SMS_ENABLED'] === 'true'`. Mask sensitive fields (token, password, apiKey) for display — return only first 4 chars + '***' if value exists, empty string if not. Add a `hasSavedValue` boolean for each sensitive field so UI knows whether a value is stored.

**`saveIntegrationSettings(type: string, values: Record<string, string>)`**:
- `await verifySession()`
- Validate `type` is one of: 'sms', 'whatsapp', 'ftp', 'telegram', 'llm'
- Map the form values to env key names (e.g., for sms: token -> SMS_TOKEN, fromName -> SMS_FROM_NAME, enabled -> SMS_ENABLED)
- Call `writeEnvValues(envMap)`
- `revalidatePath('/admin/settings')`
- Return `{ success: true }` or `{ success: false, error: string }`
- SECURITY: Do NOT write any of these values to audit_log (secrets must not be logged). Only log the action: "Settings updated for {type}" via console.info.

**`testSmsConnection()`**: `await verifySession()`. Read SMS_TOKEN from process.env. If empty, return `{ ok: false, message: 'אין token מוגדר' }`. Try to call Micropay balance/verify endpoint. If that's not available, use the standard SMS API with a clearly fake number `0500000000` and check if the response is a numeric ID (success) or error text. Return `{ ok: boolean, message: string }`. Use the Micropay endpoint from RESEARCH.md: `http://www.micropay.co.il/ExtApi/ScheduleSms.php`.

**`testWhatsAppConnection()`**: `await verifySession()`. Read WHATSAPP_ACCESS_TOKEN and WHATSAPP_PHONE_NUMBER_ID. Call `GET https://graph.facebook.com/v18.0/{phoneNumberId}?access_token={token}` to verify credentials. Return `{ ok, message }`.

**`testFtpConnection()`**: `await verifySession()`. Since basic-ftp/ssh2-sftp-client are not installed and we don't want new dependencies, do a simple TCP connection test using Node.js `net` module: `const socket = net.createConnection({ host, port }, () => { socket.end(); resolve({ ok: true, message: 'חיבור תקין' }) })`. Add a 5-second timeout. If connection refused or timeout, return `{ ok: false, message: 'לא ניתן להתחבר' }`.

**`testTelegramConnection()`**: `await verifySession()`. Call `GET https://api.telegram.org/bot{token}/getMe`. If 200 and result.ok, return `{ ok: true, message: 'Bot מחובר: ' + result.username }`. Otherwise return error.

**`testLlmConnection()`**: `await verifySession()`. Read LLM_PROVIDER, LLM_API_KEY, LLM_BASE_URL. For 'openai': `GET https://api.openai.com/v1/models` (or LLM_BASE_URL + '/v1/models') with `Authorization: Bearer {key}`. For 'gemini': `GET https://generativelanguage.googleapis.com/v1beta/models?key={key}`. Return `{ ok, message }` based on HTTP status.

**3. Each settings component (`SmsSettings`, `WhatsAppSettings`, `FtpSettings`, `TelegramSettings`, `LlmSettings`)**

All follow the same pattern:
- `'use client'`
- Props: `{ settings: { enabled, ...fields, hasSavedValue_* } }`
- Local state via `useState` for each field, initialized from props
- Enable/disable: `<Switch>` component at the top, bound to local `enabled` state
- Fields: `<Input>` for each field (use `type="password"` for tokens, keys, passwords). Show placeholder "****" if `hasSavedValue` is true and the field is empty (indicating a stored value that was masked). Include a "הצג" toggle button to show/hide password fields.
- Save button: calls `saveIntegrationSettings(type, values)` Server Action via `startTransition`. Show loading state. On success, `toast.success('ההגדרות נשמרו')`. On error, `toast.error(result.error)`.
- Test connection button: calls the appropriate `test*Connection()` Server Action. Show loading spinner. On success, `toast.success(result.message)`. On failure, `toast.error(result.message)`.
- Layout: form fields in a vertical stack with labels above each field (Hebrew labels). Save and Test buttons in a horizontal row at the bottom.
- When `enabled` is false, form fields should be visually muted (opacity-50) but still editable (user might want to configure before enabling).

**Specific fields per integration (from RESEARCH.md Integration Fields Reference):**

**SmsSettings:** Token (password), From Name (text, max 11 chars)
**WhatsAppSettings:** Access Token (password), Phone Number ID (text), Business Account ID (text, optional)
**FtpSettings:** Protocol (Select: FTP/SFTP), Host (text), Port (number, default 21), Username (text), Password (password), Remote Path (text, default "/")
**TelegramSettings:** Bot Token (password), Chat ID (text)
**LlmSettings:** Provider (Select: openai/gemini), Model Name (text, placeholder "gpt-4o"), API Key (password), Base URL (text, optional, placeholder "https://api.openai.com")

**Important:**
- When saving, if a password field is empty AND `hasSavedValue` is true, do NOT include it in the save payload (preserve the existing value). Only overwrite when the user types a new value.
- Use `useTransition` + `startTransition` for Server Action calls (React 19 pattern used throughout codebase)
- All labels and placeholder text in Hebrew where applicable
