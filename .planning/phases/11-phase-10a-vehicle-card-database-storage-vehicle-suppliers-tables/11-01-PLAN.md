**RLS policies** — Enable RLS on all 6 tables. For each soft-deletable table (vehicle_suppliers, vehicles, vehicle_tests, vehicle_insurance, vehicle_documents):
- SELECT: `USING (deleted_at IS NULL)` — hides soft-deleted rows
- INSERT: `WITH CHECK (true)` — authenticated can insert
- UPDATE: `USING (true) WITH CHECK (true)` — needed for soft-delete RPCs and normal updates

For vehicle_document_names (no soft-delete):
- SELECT: `USING (true)` — any authenticated user
- INSERT: `WITH CHECK (true)` — authenticated can insert
- UPDATE: `USING (true) WITH CHECK (true)` — for usage_count increment

Use `DROP POLICY IF EXISTS` + `CREATE POLICY` pattern. Policy naming: `{table}_{action}` (e.g., `vehicles_select`, `vehicles_insert`, `vehicles_update`).

**Computed status views:**

1. `vehicle_computed_status` — CREATE OR REPLACE VIEW with `security_invoker = true`:
   ```sql
   SELECT v.id,
     CASE
       WHEN v.deleted_at IS NOT NULL THEN 'inactive'
       WHEN v.is_active = FALSE THEN 'inactive'
       ELSE 'active'
     END AS computed_status
   FROM vehicles v;
   ```

2. **UPDATE `driver_computed_status`** — CREATE OR REPLACE VIEW (replaces existing from 00018/00023). Add vehicle assignment condition:

   ```sql
   WHEN (SELECT COUNT(*) FROM vehicles veh WHERE veh.assigned_driver_id = d.id AND veh.deleted_at IS NULL AND veh.is_active = TRUE) > 0 THEN 'active'
   ```
