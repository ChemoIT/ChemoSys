# ChemoSys — מערכת ניהול פנימית לחמו אהרון

## What This Is

מערכת ניהול פנימית מבוססת ווב לחברת חמו אהרון בע"מ (תשתיות אנרגיה). המערכת מודולרית ונבנית בהדרגתיות — מתחילים מממשק אדמין ומסך כניסה, ובהמשך מוסיפים מודולי ניהול צי רכב, צמ"ה, ומודולים עסקיים נוספים. המערכת בעברית, מלאת תמיכה במובייל וטאבלט, מאוחסנת בענן.

## Core Value

ממשק אדמין שמאפשר לנהל עובדים, יוזרים, חברות, פרויקטים והרשאות — התשתית שעליה כל המודולים העתידיים נבנים. אם האדמין לא עובד, שום דבר לא עובד.

### ממשק אדמין — פירוט טאבים

#### Tab 0: דשבורד
- סיכומים: עובדים פעילים, פרויקטים, יוזרים
- פעילות אחרונה מתוך Audit Log

#### Tab 1: ניהול חברות
- שדות: שם חברה, מספר חברה (פנימי, unique), מס' ח.פ., שם אחראי, מייל אחראי, הערות
- CRUD + Soft Delete

#### Tab 2: ניהול מחלקות
- היררכי: מחלקה + תת-מחלקה
- שדות: מספר מחלקה, שם מחלקה, מחלקת-אב, חברה, הערות
- CRUD + Soft Delete

#### Tab 3: תגיות תפקיד
- שדות: שם תג, תיאור, הערות
- CRUD + Soft Delete
- משמש לבחירה מרובה בכרטיס עובד

#### Tab 4: ניהול עובדים
- **מפתח ייחודי: מספר עובד + חברה** (composite)

- ייבוא Excel: הצלבה לפי מספר עובד + חברה → עדכון אוטומטי

#### Tab 5: ניהול יוזרים
- יוזר מקושר לעובד פעיל (חיפוש לפי שם/ת.ז./מייל/מספר עובד)
- אימות: Supabase Auth (מייל + סיסמה)
- הרשאות גמישות: לכל מודול ותת-מודול — אין גישה / קריאה / קריאה+כתיבה
- אפשרות שיוך תבנית הרשאות + דריסה ידנית

#### Tab 6: תבניות הרשאות
- שם תבנית, תיאור, סט הרשאות מוגדר מראש
- כשמשייכים תבנית ליוזר — ממלאת את ההרשאות
- יוזר יכול לדרוס הרשאות ספציפיות אחרי שיוך תבנית

#### Tab 7: ניהול פרויקטים
- שדות: שם פרויקט, שם תצוגה, מספר פרויקט (אוטומטי PR25XXXXXX), מספר הוצאה, מס' כללי, תיאור, קוד פרויקט, קוד שעון נוכחות (+checkbox), סוג (פרויקט/שטח התארגנות/שטח אחסנה), התעלם משיוך צמ"ה אוטו', פיקוח, מזמין
- מנהל פרויקט (FK → עובדים) + מייל + טלפון + הודעות
- מנהל עבודה (FK → עובדים) + מייל + טלפון + הודעות
- אחראי רכבי מחנה (FK → עובדים) + טלפון
- סטטוס (פעיל/לא פעיל), קואורדינטות (lat/lng) + "צפה במפה"

#### Tab 8: הגדרות מערכת
- עורך Config.ini (קריאה/כתיבה של קובץ פיזי בשרת)
- ניהול התממשקויות API: SMS (Micropay), WhatsApp, FTP
- כל התממשקות: שם, סוג, endpoint, פרמטרים (key-value), פעיל/לא פעיל
- הכנה לאזורי הגדרות עתידיים

#### Audit Log (יומן פעולות)
- תיעוד כל שינוי: מי, מה, מתי, ערך ישן, ערך חדש
- צפייה עם פילטרים: לפי יוזר, ישות, טווח תאריכים

### DB Schema כללי
- כל טבלה כוללת: `id`, `created_at`, `updated_at`, `created_by`, `updated_by`, `deleted_at` (soft delete)
- Composite unique: employees(employee_number, company_id)
- FKs: employees→companies, employees→departments, users→employees, projects→employees (managers)
- Junction tables: employee_role_tags, user_permissions, template_permissions

## Constraints

- **Tech stack:** Next.js + Supabase — נקבע, לא משתנה
- **Hosting:** Vercel + ch-ah.info domain
- **Language:** עברית ראשית, RTL — לא ניתן לדלג
- **Extensibility:** כל רכיב חייב לתמוך בהרחבה עתידית (מודולים, טאבים, הגדרות)
- **Soft Delete:** בכל המערכת — לא למחוק נתונים לעולם
- **Mobile:** תמיכה מלאה — לא "nice to have"
